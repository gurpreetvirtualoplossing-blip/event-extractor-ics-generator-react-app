# PHP Backend Setup Instructions

## Prerequisites

- PHP 7.4 or higher
- Composer (PHP package manager)
- Apache/Nginx web server with mod_rewrite enabled (for Apache)
- cURL extension enabled
- (Optional) poppler-utils for PDF processing (alternative to Composer package) 

## Installation Steps

### 1. Install PHP Dependencies

```bash
composer install
```

This will install:
- `smalot/pdfparser` - For PDF text extraction

**Note:** If you don't have Composer installed, you can:
- Download from https://getcomposer.org/
- Or use the system package manager:
  ```bash
  # Ubuntu/Debian
  sudo apt-get install composer
  
  # macOS
  brew install composer
  ```

### 2. Configure Environment Variables

Copy `.env.example` to `.env` and add your OpenAI API key:

```bash
cp .env.example .env
```

Then edit `.env` and set your OpenAI API key:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

**Important:** Never commit your `.env` file to version control!

### 3. Server Configuration

#### For Apache:

1. Make sure `mod_rewrite` is enabled:
   ```bash
   sudo a2enmod rewrite
   sudo systemctl restart apache2
   ```

2. The `.htaccess` file should handle routing automatically.

#### For Nginx:

Add this to your nginx config in the `server` block:

```nginx
location /api {
    try_files $uri $uri/ /api/index.php?$query_string;
}

location /health {
    try_files $uri $uri/ /api/index.php?$query_string;
}
```

### 4. PHP Configuration

Ensure your `php.ini` has these settings (the `.htaccess` file also sets these, but server-level is recommended):

```ini
upload_max_filesize = 50M
post_max_size = 50M
memory_limit = 256M
max_execution_time = 300
max_input_time = 300
```

To check your PHP configuration:
```bash
php -i | grep upload_max_filesize
php -i | grep post_max_size
```

### 5. Deploy Files

Upload these files/folders to your PHP server:

```
your-php-server/
├── api/
│   ├── index.php
│   ├── openaiService.php
│   ├── pdfService.php
│   └── config.php
├── .htaccess (for Apache)
├── composer.json
├── composer.lock (generated by composer install)
├── vendor/ (generated by composer install)
└── .env (your API keys - DO NOT commit to git)
```

### 6. Update Frontend Configuration

In your frontend `.env` file or `.env.local`, update the API URL:

**For production:**
```env
VITE_API_URL=https://your-php-server.com/api
```

**For local development:**
```env
VITE_API_URL=http://localhost/api
```

Or if running PHP in a subdirectory:
```env
VITE_API_URL=http://localhost/your-project/api
```

## Troubleshooting

### PDF Processing Not Working

If you get errors about PDF processing:

1. **Install Composer package (Recommended):**
   ```bash
   composer require smalot/pdfparser
   ```

2. **Or install poppler-utils** (alternative method):
   ```bash
   # Ubuntu/Debian
   sudo apt-get install poppler-utils
   
   # CentOS/RHEL
   sudo yum install poppler-utils
   
   # macOS
   brew install poppler
   
   # Windows
   # Download from: https://github.com/oschwartz10612/poppler-windows/releases
   ```

### CORS Issues

The CORS headers are already set in `api/index.php`. If you still have issues:

1. Check that the frontend URL is allowed in the server configuration
2. Verify the `Access-Control-Allow-Origin` header is being sent (check browser dev tools)

### Large File Uploads

If large images fail to upload, check:

1. **PHP limits** - Check current values:
   ```bash
   php -i | grep upload_max_filesize
   php -i | grep post_max_size
   ```

2. **Increase limits** in `.htaccess` or `php.ini`:
   - `upload_max_filesize = 50M`
   - `post_max_size = 50M`
   - `memory_limit = 256M`

3. **Nginx users**: Also check `client_max_body_size` in nginx config

### API Key Not Found Error

1. Verify `.env` file exists in the root directory
2. Check that the file contains `OPENAI_API_KEY=your_key_here`
3. Ensure the file is readable by the web server
4. Try setting it as an environment variable on the server

### 404 Not Found for API Endpoints

1. **Apache**: Ensure `mod_rewrite` is enabled
2. **Check URL structure**: Make sure you're accessing `/api/extract/from-text` not `/api/index.php/extract/from-text`
3. **Check `.htaccess`**: Ensure the file is in the root directory and readable
4. **Nginx**: Make sure the rewrite rules are properly configured

### cURL Errors

If you get cURL errors:

1. Verify PHP cURL extension is installed:
   ```bash
   php -m | grep curl
   ```

2. Install if missing:
   ```bash
   # Ubuntu/Debian
   sudo apt-get install php-curl
   
   # macOS
   brew install php-curl
   ```

## API Endpoints

All endpoints match the Node.js backend API exactly:

- `GET /api/health` or `GET /health` - Health check
- `POST /api/extract/from-files` - Extract events from files (images, PDFs)
- `POST /api/extract/from-text` - Extract events from text
- `POST /api/extract/from-url` - Extract events from URL (currently throws error - same as Node.js version)

### Request/Response Format

All endpoints accept JSON and return JSON:

**Request:**
```json
{
  "files": [...],  // for from-files
  "text": "...",   // for from-text
  "url": "..."     // for from-url
}
```

**Response:**
```json
{
  "events": [
    {
      "title": "Event Title",
      "startDate": "2024-10-15T14:00:00",
      "endDate": "2024-10-15T15:00:00",
      "location": "Location",
      "description": "Description"
    }
  ]
}
```

**Error Response:**
```json
{
  "error": "Error message here"
}
```

## Migration from Node.js

The PHP backend is a complete replacement for the Node.js backend. After setting up:

1. ✅ All API endpoints work the same
2. ✅ Same request/response formats
3. ✅ Same error handling
4. ✅ Same OpenAI integration
5. ✅ Same PDF processing (with PHP libraries)

You only need to:
1. Update `VITE_API_URL` in your frontend `.env` file
2. Stop the Node.js server
3. Use the PHP server instead

The frontend code does not need any changes!
